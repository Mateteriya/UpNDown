# Офлайн-партия в облаке (Supabase)

Как матч попадает в историю на **всех устройствах** под одним логином.

## Уже работает в коде?

**Да**, если выполнены все условия ниже. Иначе запись тихо не создаётся (или RPC вернёт ошибку).

## Условия (все обязательны)

| # | Условие | Проверка |
|---|---------|----------|
| 1 | В `.env` заданы `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY` | В меню есть вход Google/GitHub, не заглушка |
| 2 | На проекте Supabase применена миграция `20250406183000_record_offline_match.sql` | В SQL Editor: `select proname from pg_proc where proname = 'record_offline_match';` |
| 3 | Вы **вошли в аккаунт** до окончания партии | В шапке/меню виден профиль, не «гость» |
| 4 | Партия **офлайн** (не онлайн-комната) | Обычная игра с ботами |
| 5 | Партия **доиграна** до конца (28-я раздача) | Появился экран «Партия завершена» |

## Что происходит в коде

1. При завершении 28-й раздачи (`GameTable.tsx`) вызывается `recordOfflineMatchFinish(snapshot, имя)`.
2. Функция вызывает RPC Supabase `record_offline_match` от имени **текущего** `auth.uid()`.
3. В БД создаётся строка в `matches` (`is_offline = true`, код вида `OFF-…`) и в `match_players` (очки, место, точность заказов).
4. В «Ваш рейтинг» список подгружается через `getMyMatchHistory(user.id)`.

**Фишки и режим подсчёта в облако пока не пишутся** — только очки, место, число раздач, точность. Локальный журнал на устройстве (`partyHistory`) хранит фишки.

## Как проверить у себя

1. Войти в Google/GitHub в приложении.
2. Сыграть офлайн-партию до конца.
3. Меню → **Ваш рейтинг** → блок **«История в аккаунте (облако)»** — должна появиться строка с датой и `офлайн`.
4. На другом устройстве — тот же логин → тот же список (после обновления модалки рейтинга).

На экране **«Итоги партии»** (Подробнее) при успешной записи показывается метка «Сохранено в аккаунт».

## Если не появляется

- Не были залогинены во время финиша → запись не отправляется.
- Миграция не накатана на ваш Supabase → ошибка RPC (в консоли браузера F12).
- Другой Supabase-проект в `.env`, чем на проде → история уходит «не туда».

## Отличие от локального рейтинга

| | Локально (`profileId`) | Облако (`user.id`) |
|--|------------------------|---------------------|
| Где | `localStorage` на устройстве | Supabase |
| Нужен вход | Нет | Да |
| Другое устройство | Только если тот же `profileId` подтянулся с сервера при входе | Да, по email/OAuth |
