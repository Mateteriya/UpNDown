## Android (и потом iOS): не переписывать, а обернуть

У вас уже установлена Android Studio — это пригодится для подписи, отладки и сборки. Но **само приложение логичнее не писать на Kotlin с нуля**, а взять текущий проект (React + Vite) и упаковать его в нативное приложение:

- **Один код** — та же игра, те же обновления.
- **Сборка APK** — инструмент создаёт проект под Android (и при желании iOS), вы в Android Studio только подписываете и собираетесь.

Самый удобный вариант для Vite/React — **Capacitor** (от команды Ionic):

- Добавляется в проект как зависимость.
- Сборка: `npm run build` → затем `npx cap add android` (и при желании `ios`) → открываете проект в Android Studio и собираете APK/AAB.
- Внутри — WebView с вашим уже собранным сайтом; при необходимости доступ к нативным API (камера, уведомления, хранилище).

Альтернатива — **Cordova**, но Capacitor проще интегрировать с современным фронтом и обновлять.

**Итого по мобильным:**  
План такой: **браузерная ПК-версия + браузерная мобильная (и PWA) как сейчас, плюс APK для Android (и позже IPA для iOS) через Capacitor** (или аналог). Android Studio используется для открытия сгенерированного Android-проекта, подписи и сборки, а не для написания всего приложения на Kotlin.

## Tauri (или Electron): нужен ли отдельный десктоп-клиент?

**Tauri** — это фреймворк для десктоп-приложений: ваша веб-вёрстка (HTML/JS/CSS) в окне, бэкенд на Rust, маленький размер и хорошая производительность.

**Нужен ли он вам сейчас — зависит от цели:**

- **Если ПК-пользователи заходят в игру через браузер (или ставить PWA на рабочий стол)** — отдельное десктоп-приложение **не обязательно**. Остаётся: веб (ПК + мобильный) + PWA + APK (и позже IPA).
- **Если хотите раздавать установщик под Windows/macOS/Linux** (например, в Steam, или «скачай exe и играй без браузера») — тогда да, имеет смысл добавить сборку через **Tauri** (или, как вариант, Electron; Tauri легче и современнее).

Рекомендация: **сначала не трогать Tauri.** Сделать веб + PWA + APK (и при желании iOS). Если позже появятся запросы на «официальное десктоп-приложение» или дистрибуцию вне браузера — добавить Tauri одной целевой платформой к тому же коду.

## Итоговая схема платформ

1. **Веб (ПК и мобильный)** — как сейчас, один и тот же билд.
2. **PWA** — уже есть; при необходимости донастроить (иконки, офлайн, обновления).
3. **Android (APK)** — Capacitor (или аналог) + Android Studio для сборки и подписи.
4. **iOS (IPA)** — позже, тот же Capacitor + Xcode (нужен Mac или облачная сборка).
5. **Десктоп (exe/dmg)** — по желанию, позже, через Tauri (или Electron), если понадобится отдельная установочная версия для ПК.

Tauri не обязателен; без него у вас полноценно: браузерная ПК-версия, браузерная мобильная, PWA и APK (потом и iOS). Десктоп-приложение — опция «на потом», если будет запрос.
