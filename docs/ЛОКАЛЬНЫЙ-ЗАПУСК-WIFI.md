# Локальная игра по Wi‑Fi (LAN)

Сколько терминалов нужно — **ровно 2** на ПК-хосте. Остальное закрыть.

## Терминал 1 — игровой сервер (WebSocket)

```powershell
cd D:\Projects\UpNDown
npm run server:dev
```

Должно появиться что-то вроде `ws://0.0.0.0:3001`. **Не закрывайте** этот терминал.

## Терминал 2 — сайт игры (Vite)

```powershell
cd D:\Projects\UpNDown
npm run dev:host
```

Откроется `http://localhost:5173` (или **5174**, если 5173 уже занят «обычным» dev — смотрите строку `Local:` в консоли). **Не закрывайте** этот терминал.

## Файл `.env.local` (один раз, в корне репо)

```env
VITE_ONLINE_TRANSPORT=ws
VITE_WS_URL=ws://ВАШ_IP:3001
```

`ВАШ_IP` — IPv4 ПК в Wi‑Fi (`ipconfig`, строка «IPv4»).  
На телефоне в браузере: `http://ВАШ_IP:5173` (не localhost).

После правки `.env.local` — **перезапустите терминал 2** (`Ctrl+C`, снова `npm run dev:host`).

## Зал столов (список комнат)

В LAN комната по умолчанию **приватная** (`room_kind: private`) — её **нет** в публичном зале. Это нормально: гости входят **по коду**, который показывает хост в лобби.

Чтобы комната была в зале, при создании включите «Публичная комната» (если есть такой переключатель).

## Что НЕ нужно для LAN

| Не запускать | Зачем |
|--------------|--------|
| `npm run handbook:dev` | Портал документации, не игра |
| Второй `server:dev` | Дубликат — тормоза и путаница |
| Второй `dev` / `dev:host` | Дубликат |
| Supabase / VPN | Для партии по Wi‑Fi не обязательны |

## Проверка

1. ПК: Онлайн → создать комнату.  
2. Телефон (тот же Wi‑Fi): тот же адрес `http://IP:5173` → войти по коду.  
3. Сменить имя в меню под своим аватаром → у второго игрока имя в лобби и за столом должно обновиться.

4. **ИИ ходит только у создателя комнаты (хоста)** — тот, кто нажал «Создать комнату» на этом ПК.

После обновления кода: **выйти из комнаты и зайти по коду заново** (старые комнаты могли создаться с другим id).

## Вернуть облако (прод)

В `.env.local` закомментируйте или удалите строки `VITE_ONLINE_TRANSPORT` и `VITE_WS_URL`, перезапустите `npm run dev`.
