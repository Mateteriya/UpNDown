# План: режимы итога партии (фишки)

Документ для внедрения трёх способов подсчёта **поверх** очков раздач (`calculateDealPoints`).

| Режим | ID | Где по умолчанию |
|-------|-----|------------------|
| Как сейчас (только очки) | `points_only` | legacy / отладка |
| Середина стола | `vs_average` | «Доп. правила» casual |
| **Точный заказ** | `accuracy_bonus` | **Обычная** офлайн и онлайн |
| Банк (турнир) | `prize_pool` | Турнир / платная комната |

**Рейтинг** — по **очкам партии** (как сейчас). **Фишки** — для итогов, «денег» и турниров.

## 2. Архитектура (один источник правды)

```
Раздачи → calculateDealPoints → dealHistory + player.score
                                      ↓
                    computePartySettlement(dealHistory, mode, N игроков)
                                      ↓
              { rawScore, chips, place, inProfit, isChipWinner }[]
```

- Расчёт **детерминированный** — онлайн: все клиенты считают локально из `dealHistory`.
- В `GameState` / комнате хранится только **`settlementMode`** (+ для банка: `buyIn`, доли по N).

  └─ без изменений (только очки)

  └─ [фаза D] блок фишек в футере после конца партии

  └─ [фаза C] колонка фишек, пояснения, в плюсе / ★

  └─ [фаза F] фишки в карточке; рейтинг = очки
