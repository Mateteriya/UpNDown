# Защита мобильной версии при правках карт на ПК

Чтобы правки отображения карт в **ПК-версии** строго не задевали **мобильную**:

## 1. CSS (`src/index.css`)

- **Мобильные стили карт** — только внутри `@media (max-width: 1024px)` и при необходимости с селектором `.game-table-root.viewport-mobile`. Не менять их при доработке ПК.
- **ПК-стили карт** — только внутри `@media (min-width: 1025px)`.
- Для **двойной защиты** все новые/изменённые правила, касающиеся карт на ПК, вешать на контейнер **без** мобильного класса:
  ```css
  @media (min-width: 1025px) {
    .game-table-root:not(.viewport-mobile) .card-... { ... }
  }
  ```
  Тогда при ширине &lt; 1025px блок не применится, а при ширине ≥ 1025px у корня нет класса `viewport-mobile`, и стили применятся только к ПК-верстке.

- **Не добавлять** общие правила для карт (без медиазапроса), которые меняют размеры/оформление: они затронут и мобильную версию.

## 2. Разметка и логика (React)

- В `GameTable.tsx` и `CardView.tsx` уже есть флаги `isMobileOrTablet` / `compactMode` / `tableCardMobile` и т.п.
- Все **новые** правки, касающиеся только ПК (размер карт, сетка руки, слоты на столе и т.д.), делать в ветках вида:
  - `if (!isMobileOrTablet) { ... }` или
  - передавать в `CardView` пропы, которые на мобиле не используются (например `showDesktopFaceIndices`, масштабы через `scale`/`contentScale`).
- **Не менять** поведение или пропы, которые используются в мобильной версии (например `tableCardMobile`, `biddingHighlightMobile`), без отдельной проверки на мобиле.

## 3. Контроль

- После любых правок карт для ПК проверять отображение на ширине &lt; 1025px (мобильный вид) — ничего не должно измениться.
- Breakpoint: **1024px** — мобильная версия, **1025px и выше** — ПК.
