# Порядок хода и расположение игроков

Незыблемое правило: **следующий ход всегда у игрока по часовой стрелке от текущего**. У всех участников один и тот же игрок всегда в одном и том же относительном месте (слева/справа/напротив).

## Канонический порядок (движок)

- **Индексы:** 0 = Юг, 1 = Север, 2 = Запад, 3 = Восток.
- **Вид сверху:** Юг внизу, Север вверху, Запад слева, Восток справа.
- **Следующий по часовой:** Юг → Запад → Север → Восток → Юг.

В коде это задаётся одной константой (должна совпадать в `GameEngine.ts` и `rotateState.ts`):

```ts
const NEXT_PLAYER_LEFT = [2, 3, 1, 0]; // nextPlayerLeft(i) = следующий по часовой от i
```

То есть: от 0 (Юг) следующий — 2 (Запад), от 2 (Запад) — 1 (Север), от 1 (Север) — 3 (Восток), от 3 (Восток) — 0 (Юг).

## Где используется

- **GameEngine:** смена хода в `placeBid` и `playCard` — всегда `nextPlayerLeft(currentPlayerIndex)`; первый заказывающий/игрок в раздаче — `nextPlayerLeft(dealerIndex)`; сдающий в следующей раздаче — `nextPlayerLeft(dealerIndex)`.
- **rotateState:** порядок отображения «я, напротив, слева, справа» строится из той же геометрии: «слева» = `NEXT_PLAYER_LEFT[mySlot]`, «справа» = `NEXT_PLAYER_RIGHT[mySlot]`.
- **GameTable:** для отображения хода используется `state.currentPlayerIndex` уже в **display**-координатах (после `rotateStateForPlayer`), поэтому подсвечивается правильный слот (0 = я, 2 = слева, 3 = справа, 1 = напротив).

## Итог

- Порядок хода **один на всех**: по часовой стрелке в канонических координатах.
- У каждого клиента **слоты слева/справа** показывают одних и тех же игроков: тот, кто у меня слева, у соседа справа отображается как я и т.д.
- Менять порядок хода или геометрию стола нельзя без правок в **обоих** местах: `GameEngine` (NEXT_PLAYER_LEFT) и `rotateState` (NEXT_PLAYER_LEFT / NEXT_PLAYER_RIGHT / displayOrder).
