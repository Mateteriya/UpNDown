# Передача хода ИИ при таймауте (онлайн)

Описывает, как в онлайн-игре обрабатывается таймаут хода живого игрока и передача хода ИИ.

## Правила

1. **Минута без хода и без «Продлить»**  
   Если живой игрок 1 минуту не делает ход/заказ и **не** нажимает «Продлить на 30 сек» — за него **ходит ИИ**, фиксируется **один** таймаут, счётчик таймаутов для следующего раза сохраняется (не обнуляется).

2. **«Продлить» нажато, но за 30 с хода нет**  
   Если игрок нажал «Продлить на 30 сек» и **не** сделал ход в течение 30 с — за него **ходит ИИ**, фиксируется **первый из трёх** таймаутов.

3. **Игрок сделал ход**  
   Если игрок сделал свой ход/заказ — счётчик его таймаутов **полностью сбрасывается**, зафиксированных таймаутов нет.

4. **Три таймаута подряд**  
   Если игрок **три раза подряд** не взял игру в свои руки и за него ходил ИИ — игрок **заменяется на ИИ навсегда** (слот с `replacedUserId`). Дальше ИИ играет за него **без** паузы в 1 минуту (обычный ход ИИ по 400 мс).

5. **Возврат в игру**  
   Игрок может **вернуть себе** слот в любой момент (даже после многих раздач за ИИ) через «Вернуть управление» / reclaim.

## Предупреждение в UI

За **40 с** до истечения минуты показывается модалка «Ваш ход!» с обратным отсчётом **20 с** и кнопками:
- **Играю** — закрыть модалку.
- **Продлить (+30 сек)** — heartbeat каждые 5 с в течение 30 с; через 20 с — второе предупреждение (10 с), без кнопки «Продлить».

## Где реализовано

**Файл:** `src/ui/GameTable.tsx` — константы в начале файла, refs рядом с состоянием модалки предупреждения, эффекты и обработчики ниже по коду.

| Элемент | Что делает |
|--------|------------|
| **Константы** | `TURN_TIMEOUT_MS` (60 с), `TURN_WARNING_AT_MS` (40 с), `TURN_WARNING_COUNTDOWN_SEC` (20 с), `INACTIVE_REPLACE_THRESHOLD` (3). Presence-порог берётся из `onlineGameSupabase`: 60 с. |
| **Отслеживание «мой ход»** | Effect по `state.phase` и `state.currentPlayerIndex`: при начале своего хода пишет `myTurnStartedAtRef = Date.now()` для модалки предупреждения. |
| **Модалка предупреждения** | Effect с интервалом 1 с: при моём ходе и `elapsed >= 40 с` показывает модалку (обратный отсчёт 20 с). После «Продлить» — второе предупреждение через 20 с (обратный отсчёт 10 с), без «Продлить». |
| **Обратный отсчёт в модалке** | Отдельный effect при открытой модалке: каждую секунду уменьшает счётчик; при 0 — закрывает модалку. |
| **Хост: ход ИИ (обычный)** | Когда текущий слот — ИИ (`!userId`), хост один раз по ключу состояния делает `aiBid`/`aiPlay` и `sendState`. |
| **Хост: смена хода** | Effect по `canonicalState`: при смене `phase + currentPlayerIndex` обновляет `lastTurnChangeAtRef = Date.now()`. |
| **Хост: таймаут живого** | Effect с интервалом 10 с: если текущий игрок — живой и прошло ≥ 60 с с момента хода — проверяет presence; при `last_seen` новее 10 с — только return (не считаем таймаут, не сбрасываем счётчик). Иначе +1 к счётчику таймаутов этого слота; при счётчике ≥ 3 — `replaceInactivePlayer(idx)`; иначе один ход за игрока (ИИ) и `sendState`. Счётчик сбрасывается только при смене хода (кто только что сходил — тому слоту счётчик в 0). |

## Зависимости

- **onlineGameSupabase**: `heartbeatPresence`, `getPresence`, `DISCONNECT_THRESHOLD_MS_EXPORT` (60 с).
- **OnlineGameContext**: `replaceInactivePlayer(slotIndex)`.
